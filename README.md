# 📚 Curso de Persistencia y Arquitectura Backend con Spring Boot

Este repositorio contiene material estructurado para aprender el desarrollo de aplicaciones backend robustas, enfocándose en la gestión de bases de datos, patrones de diseño y la creación de APIs RESTful.

## 🎯 ¿Qué aprenderás aquí?

A través de estos capítulos, cubriremos todo el ciclo de vida de una aplicación moderna: desde los fundamentos teóricos y la configuración del entorno, pasando por el modelado de bases de datos con JPA/Hibernate, hasta la exposición de servicios mediante una API REST.

## 📂 Estructura del Proyecto (Índice)

Los documentos están organizados secuencialmente para guiar tu aprendizaje:

### Parte 1: Fundamentos y Persistencia
*   **[Cap01_Fundamentos](./Cap01_Fundamentos.md)**: Conceptos base sobre persistencia y entornos Java EE/Spring.
*   **[Cap02_Patrones_DAO](./Cap02_Patroness_DAO.md)**: Teoría sobre el patrón Data Access Object (DAO).
*   **[Cap03_Configuracion](./Cap03_Configuracion.md)**: Preparación del entorno de desarrollo y dependencias.
*   **[Cap04_Entidades_Mapeo](./Cap04_Entidades_Mapeo.md)**: Cómo mapear objetos Java a tablas de base de datos (JPA Entities).
*   **[Cap05_Relaciones](./Cap05_Relaciones.md)**: Manejo de relaciones (One-to-One, One-to-Many, Many-to-Many).
*   **[Cap06_Estados](./Cap06_Estados.md)**: Ciclo de vida de las entidades (Transitory, Managed, Detached, Removed).
*   **[Cap07_DAO](./Cap07_DAO.md)**: Implementación práctica de la capa de acceso a datos.
*   **[Cap08_Consultas](./Cap08_Consultas.md)**: Ejecución de consultas JPQL y nativas.

### Parte 2: Testing y Spring Boot
*   **[Cap09_Testing](./Cap09_Testing.md)**: Estrategias para probar tu persistencia y lógica.
*   **[Cap10_SpringBoot](./Cap10_SpringBoot.md)**: Migración o inicio de proyectos con el framework Spring Boot.

### Parte 3: Capa de Negocio y Servicios
*   **[Cap11_Repositorios](./Cap11_Repositorios.md)**: Uso de Spring Data JPA y interfaces Repository.
*   **[Cap12_Servicios](./Cap12_Servicios.md)**: Implementación de la capa de servicio (Business Logic Layer).

### Parte 4: Exposición de la API
*   **[Cap13_REST](./Cap13_REST.md)**: Creación de Controladores REST (GET, POST, PUT, DELETE).
*   **[Cap14_DTO](./Cap14_DTO.md)**: Uso de Data Transfer Objects para separar la entidad de la respuesta API.

## 🛠️ Tecnologías Utilizadas

*   **Java** (Versión 17 o superior recomendada)
*   **Spring Framework** (Core, Data, Web)
*   **Hibernate / Jakarta Persistence API (JPA)**
*   **BD Relacional** (MySQL, PostgreSQL u otra)

## 🚀 Cómo usar este repositorio

1.  Clona este repositorio en tu máquina local.
2.  Sigue leyendo los archivos `.md` en orden numérico.
3.  Cada capítulo incluye teoría y ejemplos de código correspondiente.
